## Reason chosen

**Axis A — per-property.** Three reinforcing reasons:

1. **Falsification gates fire per property cleanly.** Direction A/B/C lock is falsifiable against the 2026-04-27 historic-arc table (which already presumes C-shaped framing) + per-civ RN signature_creep_types (boss-tier specialization clusters at Colossal+Shielded for Greek) + cultural-sensitivity Follow-up #5 hard-gate (per-civ pools risk three-times-the-art-burden warning if B chosen). R11 variance grammar is falsifiable against §4.11.8 thresholds (does the grammar make the matchup-correctness axis non-memorizable?) — independent of which direction is locked. Cross-arc audit at RN is falsifiable against the per-civ RN matrix + per-tower RN type-coverage table + the just-locked direction + grammar. Per-property rounds keep each gate isolated; one round's failure does not destabilize the others.
2. **Cross-arc parity with per-civ + per-tower R1 axis picks.** Both prior arcs shipped 4-round queues (R1 scope + 3 content + RN). §4.7+R11 sub-pass shipping 4 rounds (R1 scope + R2 direction / R3 R11 / R4 RN) keeps the arc-shape rhythm legible across the Phase 4 spine.
3. **R11 bundling preserves arc-close cleanness.** The 2026-05-06 post-arc ratification item 1 directive bundled §4.7 + R11. Splitting R11 into a downstream sub-pass would (a) leave the R6 amendment mandate body unwritten across an arc boundary; (b) re-open §4.7 for spine-doc edits twice (once for direction lock, once for R11 body) instead of once at RN; (c) decohere the cross-arc audit (RN would have to integrate direction lock + R11 grammar even if grammar landed in a separate arc, doubling the audit work). Bundled keeps arc-close cleanness; separate would burn an extra cascade-lint + commit + dual-push cycle.

### 3x debug loop on axis

**Loop 1 — aggressive critique.** Per-property axis fragments the cross-property coherence audit. A reader auditing whether the locked direction + R11 grammar jointly defend §4.11.8 thresholds (e.g., "does Option C civ-matched bosses + random-seeded armor-tag mix produce measurable matchup-correctness?") must read R2 direction-lock + R3 R11-grammar separately, then audit jointly at R4 RN. The cross-property dependency (direction shape constrains what R11 variance can vary against — civ-matched bosses pin certain armor-tag cells per civ; shared regular pool varies the rest) surfaces as RN rework if R2 chooses A/B over C. Furthermore, the R11-grammar-independent-of-direction argument (Reason chosen #1) presumes the grammar's distributions are direction-agnostic; if the chosen direction at R2 changes the *space* over which R11 varies (e.g., Option B = three per-civ distributions), R11 grammar must be re-authored against the new space. The arc shape is brittle to a non-C R2 outcome.

**Loop 2 — steelman.** Cross-property coherence (Loop-1 critique #1) is what RN audit *exists* to deliver. Per-civ + per-tower RN both validated the per-property/per-civ-rounds + RN-audit pattern at zero cascade-violations. The direction-shape-constrains-variance-space concern (Loop-1 critique #2) has empirical force only if R2 lands non-C. R2's leading placeholder is Option C; the 2026-04-27 historic-arc table presumes C-shaped framing (round-30 antagonists are civ-matched); per-civ RN signature_creep_types cluster boss-tier at civ-distinct armor classes (Greek=Colossal+Shielded vs Aztec=Beast+Spirit+Unarmored vs Norse=Shielded+Unarmored). Three independent evidentiary streams point to Option C. R2's job is *ratifying* C (not forking to A/B) absent surprise — the leading placeholder does the heavy convergence lifting. R11 grammar authored at R3 against C-shaped direction has zero space-variance risk; if R2 forks to A/B, the arc reopens R3 properly per cascade discipline. Furthermore, the per-property axis discipline is structurally required: R11 grammar's falsification gate is §4.11.8 thresholds (memorization-meta defense), which is independent of direction shape — a memorization-meta defense at the variance level applies uniformly across A/B/C. Authoring R11 grammar in the same round as direction lock decoheres which gate is being passed.

**Loop 3 — synthesis.** Axis A is correct for *falsification-gate isolation* (each property has its own gate; one round's failure does not destabilize the others) but Loop-1 names a real cross-property coherence reader-experience cost that RN audit mitigates only at the close. Mitigation: each property round (R2-R3) closes with a brief **cross-property coherence hook paragraph** (one paragraph forward-pointing to the next round's expected dependency — e.g., R2 Direction-C lock closes with "R3 R11 variance grammar will author against C-shaped direction — civ-matched bosses pin per-civ armor-tag cells at R10/R15/R30; shared regular-wave pool variance fills the remaining 24 rounds of armor-tag mix; R3 will lock the distribution bands and per-map crystal-lock variance"). Cross-property coherence builds incrementally; RN integrates the property rounds into the cross-arc audit matrix as the integration artifact.

**Synthesis-locked output:** Axis A with embedded cross-property coherence hook paragraphs at the close of R2 / R3. RN integrates cross-arc coherence into a 3-axis audit (per-civ RN matchup_affinity ↔ per-tower RN type-coverage ↔ §4.11.8 skill-bar thresholds) against the locked direction + grammar.

Builders have no AI, no combat, no waypoint commands, single walk loop only. The §4.4 control-model body must explicitly preserve this exclusion.

This arc is **mechanical-content only** — no per-unit-archetype roster authoring, no civ-flavor identity prose, no creature-name authoring. ## Reason chosen

**Axis A — per-property.** Three reinforcing reasons:

1. **Falsification gates fire per property cleanly.** Control-model direction lock is falsifiable against the §3.4 "not full RTS per-unit micro" floor + §4.1 commander-as-summoned-ability-avatar (no controllable on-field Commander) + §4.4a Builder spec (Builder remains zero-control). Spec-grammar authoring is falsifiable against §4.11.2 (f) magnitudes (per-tier speed multipliers consistent) + the just-locked control-model (mode-flags catalog matches direction lock's mode-flags decision) + §3.4 floor at all 4 tiers. Cross-arc audit at RN is falsifiable against §4.1 + §4.4a + §4.11.2 + the just-locked direction + grammars. Per-property rounds keep each gate isolated; one round's failure does not destabilize the others.
2. **Cross-arc parity with per-civ + per-tower + §4.7+R11 R1 axis picks.** All three prior arcs shipped 4-round queues (R1 scope + 2-3 content + RN). §4.4 sub-pass shipping 4 rounds (R1 scope + R2 control-model / R3 spec-grammars / R4 RN) keeps the arc-shape rhythm legible across the Phase 4 spine.
3. **Spec-level scope discipline.** §4.4 OPEN ISSUE prose explicitly bounds the deliverable at "control model" — the per-unit roster sits downstream under Phase 5 + Follow-up #5. Per-property rounds at spec-grammar grain stay inside that scope; per-civ or per-tier rounds (Axis B/C) push into Phase 5 territory and would need to defer most authoring to placeholder, defeating the purpose. Axis D loses the falsification-gate isolation that Axis A buys.

### 3x debug loop on axis

**Loop 1 — aggressive critique.** Per-property axis fragments the cross-property coherence audit. A reader auditing whether the locked control-model direction + spec-grammars jointly defend §3.4's "not full RTS per-unit micro" floor across all 4 tiers must read R2 direction-lock + R3 spec-grammars separately, then audit jointly at R4 RN. The cross-property dependency (control-model direction's mode-flags catalog *is* the spec-grammar's mode-flag enumeration) surfaces as RN rework if R2 chooses (ii) attack-move-only or (iii) hybrid-with-tier-gated-micro instead of (i) waypoint + mode-flags — R3's pathing-mode catalog must re-author against the chosen direction. Furthermore, the spec-grammar-independent-of-direction argument presumes the engagement-rules catalog is direction-agnostic; if R2 chooses (ii) attack-move-only, the engagement-rules catalog collapses (no hold / fall-back if the only command is AM) and R3 must re-author against the collapsed space. The arc shape is brittle to a non-(i) R2 outcome.

**Loop 2 — steelman.** Cross-property coherence (Loop-1 critique #1) is what RN audit *exists* to deliver. Per-civ + per-tower + §4.7+R11 RN all validated the per-property/per-civ-rounds + RN-audit pattern at zero cascade-violations. The direction-shape-constrains-grammar-space concern (Loop-1 critique #2) has empirical force only if R2 lands non-(i). R2's prose-leading candidate is (i) waypoint + mode-flags; the §4.4 prose explicitly endorses "Waypoint + mode flags is achievable. Attack-Move command is useful" — pointing to (i)-with-AM-as-one-flag. (ii) attack-move-only loses the patrol / hold / fall-back grammar §4.4 needs-list item #3 mandates ("Engagement rules (auto-engage, hold, fall back)") — collapsing the catalog *is* the failure mode the OPEN ISSUE warns against. (iii) hybrid-with-tier-gated-micro adds T3-elite ability buttons that re-introduce on-field-hero adjacency surface (1-2 ability buttons on a controllable unit reads as "hero-lite") — risks silent §4.1 re-opening. Three independent evidentiary streams point to (i). R2's job is *ratifying* (i) (not forking to ii/iii) absent surprise — the prose-leading candidate does the heavy convergence lifting. R3 spec-grammars authored against (i) have zero space-collapse risk. Furthermore, the per-property axis discipline is structurally required: R3's falsification gate is §3.4 floor defense at all 4 tiers + §4.11.2 (f) magnitude consistency, which is independent of direction shape (the floor and magnitudes apply uniformly across i/ii/iii). Authoring spec-grammars in the same round as direction lock decoheres which gate is being passed.

**Loop 3 — synthesis.** Axis A is correct for *falsification-gate isolation* (each property has its own gate; one round's failure does not destabilize the others) but Loop-1 names a real cross-property coherence reader-experience cost that RN audit mitigates only at the close. Mitigation: each property round (R2-R3) closes with a brief **cross-property coherence hook paragraph** (one paragraph forward-pointing to the next round's expected dependency — e.g., R2 control-model-(i) lock closes with "R3 spec-grammars will author against waypoint+mode-flags direction — pathing-mode catalog enumerates {Patrol, Follow-Lane, Hold, Attack-Move} per the locked direction; engagement-rules catalog enumerates auto-engage / hold-fire / fall-back at-HP% defaults; movement-speed-per-tier reads against §4.11.2 (f) magnitudes verbatim"). Cross-property coherence builds incrementally; RN integrates the property rounds into the cross-arc audit matrix as the integration artifact.

**Synthesis-locked output:** Axis A with embedded cross-property coherence hook paragraphs at the close of R2 / R3. RN integrates cross-arc coherence into a 3-axis audit (§4.1 + §4.4a + §3.4 floor / §4.11.2 (f) magnitude consistency / control-surface tier-scaling at T1-T4) against the locked direction + grammars.
